Outil JavaScript côté client pour gérer les issues d'un projet GitHub. Tous les échanges d'information se font entre votre fureteur (le client) et Github. Une fois que cette page est chargée du serveur RoLLodeQc, aucun autre échange avec ce dernier ne sera effectué.

Utilisez un token Github personnel, aucun droit supplémentaire requis pour consulter les projets et issues.

Pour éditer les issues (les déplacer entre les milestones, etc.) il faut utiliser un token avec le droit au public_repo.
